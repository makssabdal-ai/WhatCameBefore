# AGENTS.md

Инструкции для агентов и разработчиков, которые вносят изменения в проект `WhatCameBefore`.

## Стек технологий

- Статическая HTML5-игра без сборщика.
- HTML: `index.html`.
- CSS: `styles.css`.
- JavaScript: `game.js`, чистый browser JS без npm-зависимостей.
- Локальный сервер: `server.js` на Node.js.
- Аудио: `assets/background.mp3`.
- Интеграция Яндекс Игр: через `/sdk.js` и безопасные проверки наличия SDK.
- Публикационные материалы: `yandex_publish/`.

## Структура проекта

```text
.
├── index.html                         # Основная HTML-страница игры
├── styles.css                         # Темы, адаптивная верстка, UI
├── game.js                            # Вопросы, режимы, прогресс, реклама, звук, SDK
├── server.js                          # Локальный статический сервер для проверки
├── README.md                          # Краткое описание проекта
├── AGENTS.md                          # Правила для будущих правок
├── assets/
│   └── background.mp3                 # Фоновая музыка
└── yandex_publish/
    ├── checklist_ru.txt               # Чеклист перед публикацией
    ├── description_ru.txt             # Описание для карточки игры
    ├── icons/                         # Иконки для публикации
    └── screenshots/                   # Скриншоты для публикации
```

## Команды запуска

Запуск локального сервера:

```powershell
node server.js
```

Адрес для проверки:

```text
http://127.0.0.1:4174/
```

Если порт занят, можно запустить на другом порту:

```powershell
$env:PORT='4180'; node server.js
```

## Команды проверки

Проверка синтаксиса JavaScript:

```powershell
node --check game.js
```

Проверка, что локальный сервер отдает главную страницу:

```powershell
$job = Start-Job -ScriptBlock { Set-Location 'C:\Users\ноут-макс\OneDrive\Desktop\WhatCameBefore'; node server.js }
Start-Sleep -Seconds 1
try { Invoke-WebRequest -Uri http://127.0.0.1:4174/ -UseBasicParsing | Select-Object -ExpandProperty StatusCode }
finally { Stop-Job $job; Remove-Job $job }
```

Проверка локальной заглушки SDK:

```powershell
$job = Start-Job -ScriptBlock { Set-Location 'C:\Users\ноут-макс\OneDrive\Desktop\WhatCameBefore'; node server.js }
Start-Sleep -Seconds 1
try { Invoke-WebRequest -Uri http://127.0.0.1:4174/sdk.js -UseBasicParsing | Select-Object -ExpandProperty StatusCode }
finally { Stop-Job $job; Remove-Job $job }
```

Проверка MP3:

```powershell
$job = Start-Job -ScriptBlock { Set-Location 'C:\Users\ноут-макс\OneDrive\Desktop\WhatCameBefore'; node server.js }
Start-Sleep -Seconds 1
try { (Invoke-WebRequest -Uri http://127.0.0.1:4174/assets/background.mp3 -UseBasicParsing).Headers['Content-Type'] }
finally { Stop-Job $job; Remove-Job $job }
```

## Стиль кода

- Не добавлять сборщик, фреймворк или npm-зависимости без прямой причины.
- Держать игру статической и легкой для публикации в Яндекс Играх.
- Использовать обычный JavaScript, без TypeScript и транспиляции.
- Сохранять русскоязычный интерфейс.
- Не смешивать логику, стили и HTML сильнее, чем уже сделано.
- Новые UI-состояния оформлять через CSS-классы, а не через inline-стили, кроме динамического прогресса.
- При добавлении вопросов сохранять формат `{ a, b, ya, yb, theme, fact }`.
- Правильный ответ не должен быть привязан к левой или правой колонке.
- Карьерные уровни должны использовать непересекающиеся наборы вопросов.
- Любые изменения аудио должны сохранять отдельное управление музыкой и звуковыми эффектами.
- Публикационные файлы в `yandex_publish/` обновлять вместе с изменениями, которые влияют на описание, скриншоты, архив или чеклист.

## Что можно делать

- Добавлять новые вопросы, темы и режимы игры.
- Улучшать адаптивность и визуальный стиль.
- Улучшать карту карьеры и прогрессию.
- Менять баланс очков, жизней, серий и наград.
- Улучшать интеграцию с Yandex Games SDK.
- Добавлять легкие ассеты в `assets/`, если они реально используются.
- Обновлять `yandex_publish/description_ru.txt`, скриншоты, иконки и `checklist_ru.txt`, если это нужно для публикации.
- Рефакторить `game.js`, если поведение сохраняется и проверки проходят.

## Что нельзя делать без явного согласования

- Удалять интеграцию с Yandex Games SDK.
- Удалять режимы `Карьера`, `По темам`, `Блиц`, `Ежедневный вызов`, `Бесконечный`, `Рейтинг`.
- Уменьшать базу вопросов ниже 1000.
- Удалять `assets/background.mp3`, если не добавлена замена.
- Объединять кнопки музыки и звуков обратно в одну.
- Возвращать закономерность, при которой правильный ответ чаще находится в одной колонке.
- Добавлять тяжелые библиотеки, изображения или видео, которые ухудшают загрузку.
- Ломать локальный запуск через `node server.js`.
- Менять язык интерфейса с русского на другой.
- Делать разрушительные git-команды вроде `git reset --hard` или удаления файлов без запроса.

## Правила для Яндекс Игр

- `/sdk.js` должен подключаться из `index.html`.
- Игра должна безопасно работать локально, когда SDK недоступен.
- `LoadingAPI.ready()` вызывается только после готовности игры.
- Рекламу показывать только через SDK.
- Сохранения делать через `player.setData`, но оставлять fallback на `localStorage`.
- Лидерборд использует техническое имя `bestScore`.
- Не добавлять сетевые запросы к сторонним API для вопросов или ассетов без необходимости.
- Перед публикацией сверяться с `yandex_publish/checklist_ru.txt`.

## Итоговый ответ после правок

Ответ должен быть на русском языке и коротко содержать:

- что изменено;
- какие файлы затронуты;
- какие проверки запущены;
- если что-то не удалось проверить, сказать это явно;
- команду запуска, если изменения касаются игры или локального сервера.

Пример:

```text
Готово. Изменил карту карьеры и баланс очков в game.js, обновил стили в styles.css.

Проверил:
- node --check game.js
- локальный сервер вернул 200

Запуск: node server.js
```
